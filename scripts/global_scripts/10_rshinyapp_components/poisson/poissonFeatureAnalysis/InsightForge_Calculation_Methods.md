# InsightForge Poisson 分析計算方法說明

## 概述

InsightForge 版本的 Poisson 特徵分析使用「賽道倍數」和「邊際效應」兩個關鍵指標，幫助決策者快速理解產品屬性的商業影響力。

## 核心概念

### 1. 賽道倍數 (Track Multiplier)
**定義**：從屬性最低值到最高值，銷量可能相差的倍數。

**商業意義**：
- 識別「戰略重點」- 哪些屬性是核心競爭力
- 賽道倍數越大，表示該屬性對銷量的影響範圍越廣

### 2. 邊際效應 (Marginal Effect)
**定義**：屬性每提升一個單位，銷量變化的百分比。

**商業意義**：
- 識別「日常優化重點」- 哪些屬性的小改進能帶來大效果
- 邊際效應越大，表示該屬性的單位改進效果越明顯

## 計算公式

### 方法一：使用 Poisson 迴歸係數（當有 coefficient 時）

```r
# 1. 邊際效應計算（保持不變）
if (abs(coefficient) > 5) {
  # 對於極端值，限制在 ±500%
  marginal_effect_pct = sign(coefficient) * 500
} else {
  # 標準計算：(e^β - 1) × 100%
  marginal_effect_pct = (exp(coefficient) - 1) * 100
}

# 2. 賽道倍數計算（動態範圍）
# 先獲取屬性的實際範圍
attr_range = calculate_attribute_range(predictor_name)

if (abs(coefficient) > 2) {
  # 對於大係數，使用線性增長避免指數爆炸
  track_multiplier = exp(2) * (1 + (abs(coefficient) - 2) * 0.5)
} else {
  # 改進計算：使用平方根緩和指數效應
  effective_range = min(attr_range, 10)  # 限制範圍效應
  track_multiplier = exp(abs(coefficient) * sqrt(effective_range))
}

# 最終限制在合理範圍內
track_multiplier = min(track_multiplier, 100)
```

### 方法二：使用發生率比（當只有 incidence_rate_ratio 時）

```r
# 1. 邊際效應計算（保持不變）
marginal_effect_pct = (incidence_rate_ratio - 1) * 100
# 限制範圍在 -90% 到 +500% 之間
marginal_effect_pct = max(-90, min(500, marginal_effect_pct))

# 2. 賽道倍數計算（動態範圍）
# 獲取屬性的實際範圍
attr_range = calculate_attribute_range(predictor_name)
# 使用平方根緩和指數效應
power = sqrt(attr_range)
track_multiplier = incidence_rate_ratio ^ power
# 限制最大值為 100 倍
track_multiplier = min(100, track_multiplier)
```

## 計算邏輯解釋

### 為什麼要限制數值範圍？

1. **實務考量**：
   - 真實商業場景中，很少有屬性能讓銷量相差超過 100 倍
   - 邊際效應超過 500% 通常表示模型有異常或數據有極端值

2. **避免計算問題**：
   - Poisson 迴歸的係數可能很大（如 100），直接計算 e^(4×100) 會產生天文數字
   - 這些天文數字對商業決策沒有實際意義

### 賽道倍數的動態計算

#### 2025-09 更新：動態屬性範圍計算

不再使用固定的 4 次方假設，而是根據屬性的實際特性動態計算範圍：

**屬性範圍識別規則**：
- **評分類** (rating/score/star)：範圍 = 4（1-5 星）
- **二元類** (binary/flag/is_/has_)：範圍 = 1（0-1）
- **百分比類** (percentage/percent/rate)：範圍 = 100（0-100%）
- **計數類** (count/quantity/number)：範圍 = 10（保守估計）
- **價格類** (price/cost/revenue)：範圍 = 50（中等價格範圍）
- **其他**：預設範圍 = 4

**計算公式改進**：
- 原始公式：`exp(range × |coefficient|)` 可能產生天文數字
- 改進公式：`exp(|coefficient| × sqrt(range))` 使用平方根緩和指數增長
- 範圍上限：`effective_range = min(actual_range, 10)` 避免極端值

### 實際例子（使用動態範圍計算）

1. **customer_ratings（客戶評分）**
   - coefficient = 0.5
   - 屬性範圍 = 4（1-5 星評分）
   - 邊際效應 = (e^0.5 - 1) × 100% = 64.9%
   - 賽道倍數 = e^(0.5 × √4) = e^1 = 2.7 倍
   - 解釋：評分每提升 1 星，銷量增加 64.9%；從 1 星到 5 星，銷量相差 2.7 倍

2. **has_warranty（是否有保固）**
   - coefficient = 0.8
   - 屬性範圍 = 1（二元變數 0-1）
   - 邊際效應 = (e^0.8 - 1) × 100% = 122.6%
   - 賽道倍數 = e^(0.8 × √1) = e^0.8 = 2.2 倍
   - 解釋：有保固比沒保固的銷量高 122.6%

3. **price_index（價格指數）**
   - coefficient = -0.3
   - 屬性範圍 = 50（假設價格範圍）
   - 邊際效應 = (e^-0.3 - 1) × 100% = -25.9%
   - 賽道倍數 = e^(0.3 × √10) = e^0.95 = 2.6 倍（使用有效範圍上限 10）
   - 解釋：價格指數每提升 1 單位，銷量減少 25.9%；從最低到最高價格，銷量相差 2.6 倍

## 商業決策指南

### 賽道倍數解讀
- **> 3.0 倍**：極重要因素，是核心競爭力
- **2.0 - 3.0 倍**：重要影響因素，應重點關注
- **1.2 - 2.0 倍**：有一定影響，可考慮優化
- **< 1.2 倍**：影響很小，不是關鍵因素

### 邊際效應解讀
- **> 50%**：強烈正向影響，小改進大效果
- **20% - 50%**：中等正向影響，穩定改進策略
- **-20% - 20%**：影響較小，非優先改進項目
- **< -20%**：負向影響，需要權衡成本效益

## 與傳統 Poisson 分析的差異

| 指標 | 傳統方法 | InsightForge 方法 |
|------|----------|------------------|
| 主要指標 | 發生率比（IRR）| 賽道倍數 + 邊際效應 |
| 解釋方式 | 統計顯著性 | 商業影響力 |
| 決策支援 | 學術化 | 實務化 |
| 數值範圍 | 可能很極端 | 限制在合理範圍 |

## 技術實現細節

### 數據來源
- 表格：`df_{platform}_poisson_analysis_all`
- 篩選條件：
  - `predictor_type != "time_feature"`（排除時間特徵）
  - `convergence == "converged"`（只使用收斂的模型）
  - `coefficient > 0` 且 `p_value < 0.1`（只顯示正向顯著的屬性）

### 視覺化呈現
1. **摘要卡片**：顯示賽道冠軍和邊際冠軍
2. **賽道倍數圖**：橫條圖，標示重要性閾值線
3. **邊際效應圖**：直條圖，用顏色區分影響程度
4. **策略建議**：基於分析結果的具體行動建議
5. **詳細表格**：完整數據，支援 Excel 下載

## 使用建議

1. **優先看賽道倍數**：識別戰略重點
2. **再看邊際效應**：找出快速改進機會
3. **結合兩者制定策略**：
   - 80% 資源投入高賽道倍數屬性（長期競爭力）
   - 20% 資源投入高邊際效應屬性（短期成效）

## 注意事項

1. 本方法適用於已經過 Poisson 迴歸分析的數據
2. 計算結果已考慮實務可行性，避免產生不合理的極端值
3. 建議結合其他分析方法（如定位分析）綜合評估

## 改進歷程

### 2025-09 版本改進

**問題**：
- 使用固定的 4 次方計算，導致所有屬性都假設範圍為 4
- 產生不合理的倍數（如 1560 倍）
- 沒有考慮不同屬性類型的實際範圍差異

**解決方案**：
1. **動態範圍識別**：根據屬性名稱模式判斷其可能的數值範圍
2. **計算公式優化**：使用平方根緩和指數增長，避免極端值
3. **有效範圍限制**：設置範圍上限（10），防止計算爆炸
4. **清晰的計算基礎**：在界面上顯示使用的範圍值，提高透明度

**遵循的 MAMBA 原則**：
- MP047: Functional Programming - 建立可重用的範圍計算函數
- MP081: Explicit Parameter Specification - 明確參數定義
- MP088: Immediate Feedback - 清楚顯示計算基礎
- R116: Enhanced Data Access - 正確使用數據進行計算
